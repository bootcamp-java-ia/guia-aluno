# 🚀 Demo Day — Bootcamp Java & IA

O Demo Day é o momento final do Bootcamp Java & IA, onde cada squad apresenta
o sistema desenvolvido ao longo das semanas.

Mais do que código, este momento representa:
- aprendizado real
- trabalho em equipe
- capacidade de entregar dentro do prazo

## 🎯 Objetivo
Apresentar um sistema funcional que resolva um problema real simples, demonstrando:
- clareza na solução
- domínio do que foi desenvolvido
- capacidade de explicar decisões técnicas

> ✅ O foco **não é complexidade**.
> ✅ O foco é **funcionamento + entendimento**.

## 📦 Entrega do Projeto
Cada squad deve entregar:
- Repositório no GitHub atualizado
- README completo com: Problema · Solução · Como rodar
- GitHub Project organizado
- Lean Canvas preenchido (ver [`../templates/lean-canvas.md`](../templates/lean-canvas.md))

## 🎤 Formato da Apresentação
Cada squad escolhe uma das opções:

### Opção 1 — Apresentação ao vivo
- Até 5 minutos
- Demonstrar o sistema funcionando
- Explicar o fluxo principal

### Opção 2 — Vídeo gravado
- Até 5 minutos
- Conteúdo: Problema, Solução, Demonstração do sistema
- Será exibido no Demo Day

## 🧠 Estrutura sugerida
Ver [`../templates/apresentacao.md`](../templates/apresentacao.md).

## ✅ Critérios de Avaliação
- Sistema funcionando
- Clareza na explicação
- Domínio do que foi feito
- Simplicidade bem executada
- Trabalho em equipe

## ❌ O que NÃO será avaliado
- Complexidade desnecessária
- Uso de ferramentas não ensinadas
- Código copiado sem entendimento

## ⚠️ Regras Importantes
- O sistema deve estar funcional
- O squad deve conseguir explicar todas as partes do código
- Projetos incompletos podem ser apresentados, desde que o squad explique
  claramente o que foi feito

## 🤖 Uso de IA
✅ Você pode usar IA para:
- revisar README
- estruturar explicações
- organizar ideias

❌ Não use IA para:
- gerar o projeto final sem entendimento
- preparar apresentações que você não domina

> Se você não consegue explicar, você não aprendeu.

## ✅ Checklist Final
Antes do Demo Day, valide:
- [ ] Sistema rodando do início ao fim
- [ ] Código versionado corretamente
- [ ] README completo
- [ ] Lean Canvas preenchido
- [ ] Apresentação preparada (ao vivo ou vídeo)
- [ ] Squad alinhado para explicar o projeto

## 🎉 Mensagem Final
O objetivo do bootcamp não é criar especialistas, mas ensinar **como começar
da forma correta**.

> **Simples, funcional e bem explicado** sempre vence **complexo e incompleto**.
